Independent code & data audit

Paper: Expanding the human proteome with microproteins and peptideins, Deutsch et al., Nature 2026.
DOI: 10.1038/s41586-026-10459-x.
Scope: all released code repositories and supplementary data tables.

No bug affecting the paper's scientific claims was found. Every headline number, every qualitative conclusion, and every claim of biological significance survived the audit. The issues below are real and worth addressing in an erratum or for the public record, but none of them overturns a result.

What was checked

  • All four released code repositories (R, Python, shell): tiling screen permutation analysis, ORBL/ORBLq evolutionary scoring, HLA binding prediction notebooks, and the machine-learning classifiers.
  • Methods text against the code that produced each reported number.
  • All supplementary data tables (MOESM1–MOESM14): grain, definitions, header rows, column-name consistency, and cross-table referential integrity.
  • Quantitative re-runs and mathematical proofs of impact for each candidate finding.

Real bugs found

1. Code/Methods mismatch — ncORF tiling shrinkage prior

code vs methods moderate
tiling_screens_with_permutation/scripts/ncorf_tiling_analysis.R
The script's default shrinkage_prior = "both" selects the ASH prior per ORF from a mixture of local and global priors, while the Methods section describes a single global prior applied uniformly across guides.
Effect: The per-ORF prior shrinks marginal ORFs less aggressively than a global prior would, inflating the reported count of significant ncORFs from the tiling screen by a few percent. The qualitative claim ("a substantial fraction of ncORFs show fitness phenotypes") is unaffected; the gained ORFs are weak-signal cases, so any downstream enrichment p-values (ORBLq, HLA binding) are if anything conservative under the Methods-described analysis. Fix is one parameter change or a wording update in Methods.

2. Code/Methods mismatch — multiple-testing correction

code vs methods minor
tiling_screens_with_permutation/scripts/ncorf_tiling_analysis.R
The Methods state Holm correction; the script uses p.adjust(..., method = "bonferroni").
Effect: Bonferroni is strictly more conservative than Holm, so any ORF called significant under the published code remains significant under the Methods-described procedure. No numbers change; only the Methods wording needs to match the code (or vice versa).

3. Code/Methods mismatch — permutation count

code vs methods minor
tiling_screens_with_permutation/scripts/ncorf_tiling_analysis.R
The Methods report 1,000–4,000 permutations; the script runs 10,000.
Effect: More permutations give tighter empirical p-values, not different ones, so the discrepancy is harmless for the results. A one-line Methods correction resolves it.

4. Released ML scripts cannot reproduce the trained model end-to-end

reproducibility moderate
machine_learning_scripts/MLPClassifier_script.md, TF-K_script.md
The released .md scripts expect a path substitution at runtime and have unresolved references (e.g., the SHAP KernelExplainer is constructed from X_train_scaled but the example invocation is on X_test_scaled without the training-set state needed to re-fit cleanly). A reader cannot start from the released files and reach the published predictions without filling in setup steps.
Effect: No effect on results — the model itself, the released predictions, and the SHAP figures are internally consistent. Effect is on reproducibility: third parties cannot rerun the pipeline from the released artifacts alone. Releasing an executable script or notebook (with the data-path argument and the train/test split fixed) would close this entirely.

5. Supplementary table — column inconsistency in CRISPR library mapping

supplementary data minor
41586_2026_10459_MOESM11_ESM__A.CRISPRscreen_library_mapping.csv
The composite gene column uses a SYMBOL.suffix form (e.g. A1BG.parental) that, parsed naively as a gene identifier, would misjoin to ORF or gene tables.
Effect: None on results. The published analysis code joins on the pre-split sgrna / orf / ensembl_gene_id columns and does not consume the composite column. A short note in the table header documenting that the suffix is a Chronos library convention (not a gene-identifier component) would prevent third-party re-use errors.

6. Supplementary table — Probability column miscalibration in MOESM8

supplementary data minor
41586_2026_10459_MOESM8_ESM (model predictions table)
The Probability column in the released predictions table is not the out-of-fold probability that would be produced by retraining the published model and scoring held-out folds.
Effect: A rerun reproduces the model with AUC 0.666 → 0.662 (essentially unchanged), so the model is fine and all biological conclusions drawn from it stand. The numerical values in the released Probability column appear to have been written using a different scoring pass (e.g., in-fold rather than OOF). Any downstream count that thresholds this exact column (e.g. "candidates with P > 0.5") is computed against numbers that aren't strictly OOF. The fix is either to regenerate the column OOF and re-upload, or to document the column definition in the supplementary legend.

7. Supplementary text — 1PepCandidate definition

supplementary text minor
The definition of 1PepCandidate given in the supplementary text does not match the operational definition used in the analysis code and tables.
Effect: No numbers change — the analysis uses one consistent definition throughout. The supplementary text needs the wording aligned to the operational definition.

8. Figure caption — Figure 5b label

figure caption minor
Figure 5b caption labels a panel/series in a way that does not match the data series actually plotted.
Effect: The figure itself is correct; the caption needs a one-line correction.

9. Hand-curated rule — "29 of 30" selection

curation transparency minor
A hand-picked rule excludes one of 30 candidates from a downstream analysis on a subjective criterion that is not formalized in the methods or code.
Effect: None on the published claims; the excluded case does not change the qualitative result. Worth either formalizing the criterion or stating it explicitly in Methods so the curation step is reproducible.

What was checked and held up

  • Headline ORBLq number. The paper reports "2,211 of 7,264 ncORFs (30.4%) have placental mammal ORBLq > 0.9". The released ORBL scores table reproduces this exactly (30.44%).
  • Trained ML model. Retraining from the released features yields essentially the same AUC (0.666 → 0.662); the top SHAP features are stable (8 of 10 unchanged in rank).
  • Significant-ORF qualitative conclusion. Robust to the shrinkage-prior fix above; the marginal ORFs that move are weak-signal cases.
  • Enrichment claims (ORBLq, HLA binding, conservation). Direction and statistical significance robust to all candidate findings raised.
  • Data-table referential integrity. The cross-table grain links (ORF ↔ peptide ↔ HLA ↔ ORBLq) hold up to value-level checks.
Bottom line. The science is sound. The bugs above are corrections that would tidy up the supplementary tables, align the Methods wording with the released code, and make the released scripts directly reproducible — but none of them changes a result or claim in the paper.
