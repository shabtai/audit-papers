A reproducibility audit of DrugGPT

Released artifacts do not support the headline benchmark comparisons — a methodological synthesis grouped by validity axis
Subject paper: A collaborative large language model for drug analysis, Nature Biomedical Engineering (2025) · PMID 40987953 · PMC13190338
Scope. This audit evaluates the released code, data, and numerical claims. It does not infer author intent, misconduct, or the intrinsic quality of the underlying DrugGPT model. The conclusions are limited to whether the released artifacts support the paper's reported empirical comparisons. The audit treats the released artifacts as the object of evaluation: if the published numbers were produced by a different internal code path, preprocessing pipeline, split definition, or grader, the issue becomes one of artifact mismatch rather than necessarily numerical error. In either case, the released materials do not currently provide reproducible support for the headline comparisons.

TL;DR

The paper's three flagship comparisons — 88.2% on USMLE/MedQA vs Med-PaLM 2 (86.5) and GPT-4 (83.5); 88.0% on PubMedQA vs Med-PaLM 2 (79.2); 97.1% on Drug–Drug Interaction vs GPT-4 (62.8) — are not supported by the released code and data as currently available. Five independent methodological validity axes converge on the same conclusion: every headline number is computed on a different problem than the baseline it is compared against, on contaminated test populations in which every test row appears verbatim in the file the released harness loads as training data, and with scoring code that asymmetrically penalises any model not fine-tuned to the released harness's bare-letter output format.

The strongest single observation is structural: in the released artifacts, the file loaded as training data contains 100% of every MedQA test set (US, Mainland, Taiwan) with the correct answer attached. A central structural inconsistency is that the released scoring code, on the released data, has a mechanical accuracy ceiling of 0.31% on the US-MedQA test split — so whatever code path produced 88.2% is not reproduced by the released evaluation path.

Individually any one of these findings would be a serious methodological concern. Their combined effect is that the cross-paper accuracy comparisons in Table 1, and the multilingual and ablation claims that depend on them, are not interpretable as evidence of the proposed method's effectiveness from the released artifacts alone. These findings could be resolved by identifying an alternative evaluation path, unreleased preprocessing step, or held-out split that exactly reproduces the reported numbers and preserves comparability to the cited baselines.

1. How the audit was carried out

The audit was carried out using natural-joints data layers — semantic enrichments that give an agent a reliable, machine-readable understanding of what each column, table, and file in a dataset actually means. The enriched view of the released DrugGPT data was reconciled against the released harness code and the paper's numerical claims; a finding is recorded only where those three sources are in tension.

Scope is restricted to findings material to the paper's methodological claims. Pure reproducibility issues — missing files, broken hosting, absent preprocessing — are out of scope here: they block re-running the experiment but do not themselves shift a reported number.

2. The five methodological validity axes

Forty-six independent findings collapse onto five axes. Within each axis the findings compound: every member pushes the comparison in the same direction, and the joint effect is larger than any single member.

Axis A — Test-set contamination

What it does: Means reported accuracy on the affected splits is compatible with memorization rather than generalization, because the test rows are inside the file the released harness loads as training data, with the correct answer attached.

A.1 In the released artifacts, every MedQA test row appears in the file loaded as training data

MAJOR

Observed. The file qbank.parquet, which the released harness loads when its training flag is set, contains:

  • US MedQA: 1273 / 1273 test rows (100.0%)
  • Mainland MedQA: 3396 / 3396 test rows (100.0%)
  • Taiwan MedQA: 1413 / 1413 test rows (100.0%)
  • US MedQA train: 10176 / 10176 (100.0%)

The overlap is verbatim — same question text, same option set, same gold answer.

Implication. The reported held-out accuracy on these splits is compatible with memorization and cannot, on the released artifacts alone, be interpreted as evidence of generalization. The memorization upper bound on the reported splits is 100%.

A.2 In the released artifacts, every PubMedQA-L test PMID appears in the file used by the harness for fine-tuning, with the correct label

MAJOR

Observed. The 500-row canonical PubMedQA-L test split, on which Med-PaLM 2 reports 79.2%, is fully contained in pqal — the file the released harness reads to fine-tune. All 500 PMIDs are present; the gold label matches the test gold on 500 / 500.

Implication. The PubMedQA convention requires excluding test PMIDs from the fine-tuning split before training; the released harness performs no such filter. On these artifacts, the reported 88.0% is compatible with memorization of the labelled test PMIDs and is not comparable to baselines that follow the standard exclusion convention.

A.3 Mainland: 11% of the test set is duplicated verbatim inside the training set, and the duplicates often disagree on the gold answer

MODERATE Of 3,426 Mainland-test rows, 379 (11.2%) appear verbatim as Mainland-train rows. Of those 379 contaminated pairs, 74 (19.5%) carry a different gold answer in train than in test — direct evidence of supervision-noise on top of leakage: at least one of the two labels is wrong.

A.4 Lower-frequency label-noise findings on the same axis

  • US-MedQA train: at least 2 questions with multiple gold answers across rows (MINOR).
  • Mainland train: 5.86% duplicated question-strings (training distribution distortion).
Combined effect on the paper: On the released artifacts, the headline accuracy on US, Mainland, and Taiwan MedQA, and on PubMedQA-L, is computed on test populations that are fully contained in the files the released harness loads as training inputs. Differences between DrugGPT and the baselines on these benchmarks cannot, from the released materials alone, be attributed to better generalization, because the test sets are not held out from the released training inputs.

Axis B — Benchmark redefinition

What it does: The reported number and the baseline it is compared to are computed on different problems. The headline comparison is therefore not measuring what it appears to measure.

B.1 PubMedQA is reported on the wrong split — and answered with the wrong inputs

MAJOR Med-PaLM 2's 79.2 PubMedQA number is reported on the 1,000-row labelled split pqal (canonical test = 500). The DrugGPT paper claims "10,000 PubMedQA test samples", which can only be sourced from pqaa — the 211,269-row auto-derived split. On pqaa, the yes-rate of the auto-label is 92.8%; a model that answers "yes" every time already reaches 92.8% on this set. The reported 88.0 is therefore below the trivial majority-class baseline on its own evaluation population.

MAJOR PubMedQA is by construction an open-book task: each question ships with the supporting abstract in the column contexts (plural). The released harness's loader guards on 'context' (singular), a column that does not exist in these files. The abstract is therefore silently elided on every PubMedQA call: on the released code path, the question is answered without the supporting abstract. PubMedQA closed-book is a strictly harder task; in the literature it sits 10–15 points below open-book scores. Under the released code path the reported 88.0 would represent a substantial new closed-book result, which is not how the number is presented in the paper.

B.2 PubMedQA is silently turned from a 3-class problem into a 2-class problem

MODERATE PubMedQA-L is yes/no/maybe (552 / 338 / 110 in the labelled split — 11% maybe). The harness emits yes/no only, and rebalances those two classes to a 50/50 mix. Med-PaLM 2's 79.2 number is on the 3-class problem; DrugGPT's 88.0 is on a force-balanced 2-class problem. The random-baseline floor shifts from 33.3% to 50.0%, and the cross-paper comparison is across different problem definitions.

B.3 USMLE / MedQA "10,000 test samples" is not the canonical test set

MAJOR The canonical USMLE/MedQA test set is 1,273 rows — the population on which Med-PaLM 2 reports 86.5 and GPT-4 reports 83.5. Reaching 10,000 rows requires either concatenating train + dev + test (12,723) or pulling from qbank, which we already know contains the test set. Neither population is a "test" set in the sense that the cross-paper baselines use.

B.4 Drug–Drug Interaction is reported on a different task than its baseline

MAJOR The canonical DDI-2013 benchmark — on which GPT-4's 62.8 is reported — is a 4-class or 5-class problem (EFFECT, MECHANISM, ADVISE, INT, plus none); published 4-class state-of-the-art sits in the 70s. The released DDI evaluation discards the multi-class structure: 500 positive examples plus 500 author-synthesised negatives (no none rows exist in the released data), then scored as binary positive-vs-negative. DrugGPT's 97.1 vs GPT-4's 62.8 compares a binary task with synthesised negatives against a 4-class task on real data.

Combined effect on the paper: The three flagship comparisons against external baselines (Med-PaLM 2 on USMLE, Med-PaLM 2 on PubMedQA, GPT-4 on DDI) are each, on the released artifacts, on a different task or a different test population than the baseline number they are placed next to. Independent of any data-side issue, the headline table does not, on these artifacts, compare like with like.

Axis C — Scoring asymmetry

What it does: The grading parsers asymmetrically mis-score models that emit well-formed but non-canonical answers. Strong instruction-following models are systematically downgraded; a model fine-tuned on the released harness's exact output format escapes the bias.

C.1 On the released artifacts, the mechanical scoring ceiling for US-MedQA is 0.31%

MAJOR

Observed. In qbank the answer column is a single letter (100.0% of rows). In the canonical MedQA test file the answer column is the full answer text (single-letter rate 0.6%). The released grader compares the model's letter prediction to the gold answer field via direct equality. On the canonical US-MedQA test split, 1269 / 1273 rows (99.7%) score 0 by construction — gold is text, prediction is a letter.

Implication. The mechanical accuracy ceiling of the released evaluation path on the released data is approximately 4 / 1273 = 0.31%. The reported 88.2% is therefore not reproduced by the released evaluation path; reproducing it requires an alternative code path or label column that is not part of the public release.

C.2 The harness drops MedQA's fifth option but the data is mostly 5-option

MODERATE The MedQA prompt template lists options A–D, and the parser accepts A–D. The data, however, is dominated by five-option questions:

  • US test: 202 / 1273 (15.9%) rows have gold = E
  • US train: 1841 / 10178 (18.1%) gold = E
  • Mainland test: 633 / 3426 (18.5%) gold = E

A perfect A–D model still misses every E-row, capping achievable accuracy on US-test at ≈84% from the data side alone. qbank goes further — up to 13 options per row, with 5.49% of rows gold beyond E.

C.3 The yes/no parser maps hedge text to the wrong class — asymmetrically

MODERATE The PubMedQA parser uses substring containment. On representative hedge phrasings ("not necessarily", "probably yes", "it depends", …), 4 / 8 are mapped to "no", 1 / 8 to "yes", and 0 / 8 to "maybe". Hedging models (GPT-4, Claude) emit hedge text at 5–15% on PubMedQA-class tasks in the literature; non-hedging fine-tunes do not. Every hedge therefore moves a frontier-model prediction from "maybe" to "no" while leaving DrugGPT's bare yes/no outputs untouched.

C.4 The multiple-choice parser discards the formats strong LLMs default to

MODERATE The parser extracts the final answer with roughly last_token[0].isalpha(). The forms that survive are bare letters and "Answer: D"-style sentences. The forms that fail are:

  • (D) — first character is '(', not alphabetic → empty string returned.
  • **A** — first character is '*' → empty string returned.
  • The answer is D with no token split — first character is 'T' → returns 't', scored wrong.

These are the default formats produced by strong instruction-following models (GPT-4, Claude, Gemini) on MedQA prompts. The fine-tuned model whose training set's gold answer is itself a bare letter — DrugGPT — does not emit them and is not penalised.

C.5 The free-text grader gives free positive credit

MINOR The free-text grader is substring containment. On US-test, 7 / 1273 (0.55%) rows have the gold answer as a substring of a non-gold option in the same question (free FP credit); 20.7% of gold answers are a single token (highest substring-collision risk against any free-form prediction). Effect bounded by the small fraction.

C.6 The multilingual claim is partially trivialised at the data level

MODERATE In the Taiwanese training set, 17.8% of the nominally Chinese rows have answer_en == answer_zh — the answer was not translated. On those rows, English-token matching is sufficient to score correctly regardless of the question's language; the multilingual generalization claim, as supported by these rows, is correspondingly weaker.

Combined effect on the paper: Every parser and grader pathology in the released harness pushes accuracy in the same direction: down for any model whose outputs do not look identical to the harness's expected bare-letter format, and neutral for a model fine-tuned on that exact format. The reported gap between DrugGPT and the frontier baselines is therefore not separable from the parser asymmetry on the released artifacts, even before considering the contamination of Axis A.

Axis D — Ablation invalidity

What it does: The paper's mechanism claims — that specific augmentation steps drive the reported gains — are tested by ablations whose inputs were never actually different.

D.1 The MetaMap ablation compared two identical inputs

MODERATE The paper reports a MetaMap-augmentation ablation, comparing scores with and without MetaMap-enriched data. The "with" and "without" parquet files on the released archive are SHA-256 identical on 3 / 3 US pairs checked — same bytes, same rows, same fields. The two conditions of the ablation are the same file. The ablation cannot have measured the effect it reports, because the inputs to the two arms were not distinguishable.

D.2 The few-shot exemplar replaces the actual question on one model backend

MAJOR In the "Evidence Generation" stage on the OpenAI backend, the prompt-construction code threads the few-shot-enriched example variable (dsdg_enriched_input) but never the per-row test input (input_data). Structurally the model is asked to answer the hard-coded few-shot vignette every time, not the benchmark question. The magnitude is contingent on whether this code path was the one used for the headline numbers — the weights are not released, so we cannot rerun. If it was, accuracy on novel questions is essentially the exemplar's correctness on the hardcoded vignette: near-zero.

D.3 Schema drift between files claimed to share a schema

MODERATE The MedQA-family parquet files (qbank, test, train) share column names but not column meanings. qbank.answer is a single letter; sibling files' answer column is full answer text. Any code path that concatenates these files by column name silently mixes two different supervision encodings.

D.4 Split-membership column inverted on the entire Taiwanese training set

MODERATE 100.0% of rows in the Taiwanese training file carry meta_info == "taiwanese_test_Q" — the field that elsewhere in the pipeline distinguishes train from test. Any filter that uses meta_info to select test rows pulls the entire training corpus instead.

D.5 47% letter drift between two files of the same MedQA questions

MAJOR On 1,273 USMLE questions shared between the 4-option and 5-option parquet files, the gold-answer letter disagrees on 600 (47.1%). If the harness drew prompts from one file and gold from the other — and the schema-drift findings above show this is structurally possible — roughly 47% of grading is on a mis-labelled gold key.

Combined effect on the paper: The mechanism claims that motivate DrugGPT's design — that MetaMap augmentation helps, that evidence generation contributes, that the multi-stage pipeline behaves as its diagrams describe — rest, in the released artifacts, on ablations whose two arms are byte-identical inputs, on a code path that on one model backend structurally never reads the benchmark question, and on data files whose nominal shared schema masks divergent column meanings.

Axis E — Population sentinels and silent encoding

What it does: Sentinel values and missingness encodings are read as data, inflating sample sizes and silently changing what is being graded.
  • MINOR UNLABELLED sentinel is encoded as data: 6 hits in pqal, 4,662 hits in pqaa (2.21%). The pipeline scores them as if labelled.
  • MINOR 4.5% of the Taiwanese training set references a previous question or a figure that the corpus does not contain — structurally unanswerable rows for any image-blind model.
  • MINOR 2.7% of "English" Taiwanese rows still contain CJK token strings (translation step skipped them).
  • MINOR 0.5% of one training file has duplicate primary keys (114 affected rows over 51 keys).

None of these alone meaningfully shifts a headline number; together they place the noise floor of the released supervision a few points above zero.

4. Why the axes matter jointly, not individually

A natural reading of any single axis is contingent: perhaps the contaminated file was not the one used to train; perhaps the released scoring code was not the path that produced the number; perhaps the parser asymmetry is offset by something else. The joint structure of the findings constrains the plausible interpretations:

  • If the released code path was the one used for the headline numbers, the C.1 ceiling of 0.31% is incompatible with the reported 88.2% on the released data.
  • If a different code path was used, that path's behaviour is not part of the release and cannot be audited from public materials; whichever path it was, it was not the one tested by the released ablations (Axis D).
  • If the headline numbers are credited as valid in spite of Axis C, the Axis A contamination means they are upper-bounded by memorization rather than by learning.
  • If the contamination is set aside, the Axis B redefinitions mean the numbers are still not on the same task or population as the baselines they are placed next to.

Each axis constrains one of the available readings. The conjunction is not a chain of independent doubts — it is a methodological envelope around the set of interpretations under which the released artifacts could constitute reproducible support for the reported comparisons.

5. What this audit does not claim

  • It does not evaluate the underlying model. The weights are not part of the public release, so the model itself has not been re-run.
  • It does not infer intent. Every finding is described in terms of what the code and data do, not in terms of why.
  • It does not assess the intrinsic merit of the proposed architecture. It assesses whether the released artifacts constitute reproducible support for the numerical comparisons reported in the paper.

6. Summary

Across forty-six independent observations grouped onto five orthogonal methodological validity axes, the released code and data for DrugGPT do not, in their current form, support the comparisons reported in the paper's headline table. Specifically:

  1. In the released artifacts, the MedQA and PubMedQA-L test sets are contained at 100% coverage in the files the released harness loads for training, with the correct gold answer attached.
  2. The reported numbers are computed on different problems than the external baselines they are placed next to — different sample sizes, different class structures, closed-book vs open-book, binary vs multi-class.
  3. The released grading code has a mechanical ceiling of 0.31% on the canonical MedQA test and contains asymmetric parser pathologies that downgrade frontier baselines.
  4. The mechanism ablations were run on byte-identical inputs, and at least one stage of the pipeline structurally never reads the benchmark question on one model backend.

These issues are not corrigible by reanalysis of the released artifacts in their current form: the artifacts are themselves the object of the audit. A defensible re-evaluation of the proposed method would require:

  • A publicly specified held-out split with row-level decontamination checks against every released training file.
  • A scoring harness whose mechanical ceiling on the evaluated population exceeds the reported accuracy.
  • Benchmark definitions matching those of the cited baselines (canonical test sizes, the same class structures, open- vs closed-book parity).
  • Ablation inputs whose file hashes and row counts differ in the intended way.
  • Release of the exact preprocessing and evaluation path used to produce the headline table.

None of these conditions are met by the current public release. Any one of them being established by the authors would resolve, in part or in whole, the corresponding axis of this audit.
