Key takeaways

  • 37 unique findings; all 37 NOVEL — no GitHub issues, no Nature errata, no PubPeer thread, no Retraction Watch entry. Paper has been public 3 months.
  • 34/37 VERIFIED outright; 2 PARTIALLY VERIFIED (F-B6 impact narrower than feared; F-D3 CD68 space is trailing not leading); 1 REVISED after empirical verification on the analytic data (F-B12 — bug genuine but latent in this cohort). Six findings — F-B2, F-B3, F-B4, F-B6, F-B9, F-B12 — were re-run against the actual h5ad / CSV / code; see the Empirical verification section.
  • 4 BLOCKER code bugs, 1 CRITICAL publication-data error: F-A2 N=801/N=861 typo (off by 10× from CSV percentages); F-B1 SCLC→Squamous; F-B2 IDH2/SDHA/HEX1/NAKATPASE → empty pathway bucket; F-B9 censored landmark survival → 0 (empirically confirmed: +18.6 pp optimistic bias on OS-24mo rate, 14/55 patients censored before landmark).
  • F-B12 (K=3 CN labels): REVISED. The function relabelUsingTumourFractions is genuinely buggy — it inverts the cluster→rank mapping. Enumerating all 6 KMeans→tumor-fraction permutations: 2/6 (cyclic) produce wrong labels; 4/6 are no-ops. In this dataset's saved CSV, labels happen to be directionally correct (Tumour cluster has highest tumor-tumor intensity 183.27; Stroma has highest fibro-fibro intensity 29.98). The bug is real and reproducibility-fragile (F-C2 missing random_state compounds it), but the published Figs 4/7 are NOT mislabeled at the cell-type level.
  • F-A6 — paper Methods text inverts CD45 polarity on macrophages / endothelial / fibroblast / myofibroblast definitions. The analysis is correct (rules CSV is right); the publication prose is biologically wrong.
  • F-B5 downgraded from BLOCKER to NEGLIGIBLE — the buggy mannwhit function is only invoked in an exploratory cell with no saved output; the substantive Mann-Whitney uses a separate correct function.
  • Enrichment contribution: high. 7 ENRICHED-only findings (mostly in univariate.py + cross-file pathway disagreement) + 1 CORRECTED (the "15 vs 14 cell types" framing).
  • F-A1 — the published dataset/ ships 4 reference/summary CSVs, not the analytic data. The pipeline reads yale_*.csv + AnnData behind the DOI; the repo README does not say so.

pathways dict routes IDH2 / SDHA / HEXOKINASE1 / NAKATPASE to empty-string key — the markers are STRIPPED from the cell-type string entirely. notebooks/2-GenerateCellularNeighbourhoods.ipynb cell 36: '': ['IDH2high','IDH2','HEXOKINASE1high','HEXOKINASE1','SDHAhigh','SDHA','NAKATPASEhigh','NAKATPASE']. F-B6 BLIND MAJOR · PARTIAL (empirically narrower than feared) · NOVEL

Translation: the bug is real and shifts individual KM curves, but at the n=55 cohort level the headline p-value lands on the same side of 0.05 for ~99% of features. Headline "significant" KM features are NOT systematically created by this bug.

Impact: MINOR-MODERATE (figure-level cosmetic differences; very few significance crossings).

F-B12 BLIND MEDIUM (REVISED from MAJOR) · REVISED after empirical verification · NOVEL

K=3 CN labels: relabelUsingTumourFractions uses its mapping in the wrong direction. nolan.py:191-202 builds mapping = {i: sortedIndexByGroup[i] for i in range(K)} and applies [mapping[lab] for lab in _labels]. The CORRECT mapping is {orig_cluster: rank}; the code's mapping is its inverse permutation. Bug enumeration: of 6 possible KMeans-cluster→tumor-fraction orderings, the buggy mapping equals the correct one in 4/6 cases (identity + 3 transpositions, all self-inverse) and produces wrong labels in 2/6 cases (the cyclic permutations (2,0,1) and (1,2,0)).

Empirical verification on this dataset: the saved K=3 labels in 3-metabolic_GCross.csv are directionally correct — cluster named 'Tumour' has mean tumor-tumor G-cross intensity 183.27 (highest); 'Stroma' has highest fibroblast-fibroblast intensity 29.98 vs 1.99 in 'Tumour'; 'Tumour_Stroma_interface' is intermediate on both. KMeans for this cohort landed in one of the 4/6 self-inverse permutations, so the bug was a no-op here.

Latent risk: the bug is real and silently fragile. With F-C2 (missing random_state in the K=3 KMeans call, notebook 2 cell 45) the cluster-id permutation is non-deterministic across reruns and would land in the 2/6 buggy regime on ~33% of seeds — at which point Figs 4/7 K=3 labels WOULD be scrambled. This dataset's published figures are not mislabeled at the cell-type level; reproducing the analysis is.

Impact: MEDIUM (latent; would be MAJOR under a different KMeans seed; published figures unaffected at the cell-type level).

  2. F-B2 + F-B3 + F-B4 stack on the metabolic pathway story. Net read

After empirical verification against the analytic data: the paper's headline metabolic-neighborhood + KM-survival story is supported by code with two quantitatively-confirmed BLOCKERs — F-B2 (IDH2/SDHA/HEXOKINASE1/NAKATPASE silently stripped from the very cell_types_immune_metapath column that MetaPathNeighbourhood clustering and every Funcmetapath feature in Figs 4 / 7b are built from; the markers appear in 24–68% of pre-strip tumour-meta profiles and 0% of post-strip ones) and F-B9 (+18.6 pp inflation on OS-24mo rate). Several other substantive bugs were dampened by the n=55 cohort's specific draw (F-B6 mean-vs-median shifts only 1% of features across p=0.05; F-B12 K=3 labels are directionally correct in the saved CSV because the KMeans permutation landed in a self-inverse case) but remain reproducibility-fragile (F-C2 missing random_state + F-B12 mapping bug would scramble K=3 labels on ~33% of seeds). F-A2 is an obvious publication-data error any reader can catch. F-A6 is a biology error in the paper text but the code is correct. F-A1 means a reader cloning the repo cannot reproduce anything end-to-end without the AnnData behind DOI 10.48610/73b218c — which the README does not link.
